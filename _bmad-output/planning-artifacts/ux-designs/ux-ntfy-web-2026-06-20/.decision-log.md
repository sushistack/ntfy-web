### 2026-06-20 — Card visual language (from palette study v1→v2 + body/button studies)
Artifacts: `.working/color-themes-2.html`, `button-accents.html`, `card-body-variants.html`.
- **UX-D1 — Dark = neutral, NOT brown.** Overrides PRD D12 (warm charcoal w/ brown undertone). Cool-neutral charcoal; color character carried only by brand teal. Proposed: bg `#15171A` (standard) or `#0F1113` (deeper) — *depth still open*. Light counterpart also neutral (`#F3F4F6`/`#FFFFFF`).
- **UX-D2 — Card left edge squared.** `border-radius: 0 16px 16px 0` — priority accent bar side is not rounded; right side keeps `radius-md` 16.
- **UX-D3 — Card body is adaptive, supports all three: paragraph / key-value rows / bullet list.** Driven by message content (free text → paragraph; structured → key-value, failed=coral, ok=teal; itemized → bulleted). Jay: "A,B,C 3개 다 지원."
- **UX-D4 — Meta vs tags separated.** Meta row = topic chip (left) · relative time (far right). Tags moved to their own row (above meta). Fixes tag-after-time. 
- **UX-D5 — Chip shapes differentiated.** Top = priority = squared solid badge (radius 6, filled priority color). Bottom = topic = soft-tinted pill; tags = outlined pill (radius 999). Squared-vs-pill contrast intentional.
- **UX-D6 — Buttons achromatic.** Primary/emphasis = white fill (`~#F4F5F6`) + dark text; secondary = neutral ghost (transparent + 1px neutral border, light-gray text). Brand teal reserved for nav / links / unread dot — NOT buttons. Colored-accent button idea (indigo/violet/etc.) rejected.
- **UX-D7 — Roomier card.** Padding 20px (left 22), body line-height 1.6, larger inter-section gaps, actions separated by a top divider.

- **UX-D8 — Dark depth = standard.** _(Superseded by UX-D10 below — Beszel near-black.)_

### 2026-06-20 — Beszel-feel direction adopted
Reference: `imports/beszel-reference.md`. Mock: `.working/beszel-feel-shell.html`.
- **UX-D9 — Accent = emerald green `#42D392`.** Overrides PRD D7 (brand teal `#338574`). Per Jay, the Android sister app's brand color changes to green too (shared token stays 1:1). Green used for: active-nav dot, status/unread dot, links, meter "ok" fill, brand logo.
- **UX-D10 — Background = near-black neutral `#0C0D0F`; card surface `#16181B`, border `#23262B`.** Supersedes UX-D1/UX-D8 hexes (kept neutral, just darker per Beszel).
- **UX-D11 — Key-value rows get leading mono icons + inline meter bars** for numeric/percentage values. Meter thresholds reuse priority palette: green (ok) → amber (≥ warning, default 65%) → coral (≥ critical, default 90%). This is the richest form of UX-D3's key-value body.
- **UX-D12 — Card has a header band:** priority badge + title + trailing icon buttons (bell/mute, overflow ⋯), divider under it. Body below.
- **UX-D13 — Sidebar active state = gray surface (`#1C1F23`), NOT colored** — only the leading dot is green. Confirms achromatic chrome; green is reserved for accent moments.
- UX-D6 (white buttons) reconfirmed by Beszel's white "Save Settings".
